Product Studio

How We Work

Start with Human Behavior

Every project begins with understanding emotional needs. We map when people feel specific needs most acutely and what currently stops them from acting on those needs.

Prototype Fast with Streamlit

Before building full applications, we validate core AI features using Streamlit prototypes. This lets us test prompt chains, user interactions, and core utility quickly and cheaply. Many ideas start as Streamlit toys that prove whether people would actually be upset if they disappeared.

Build with Modern, Proven Tools

Prototyping: Streamlit for AI feature validation and standalone experiments
Frontend: Next.js with React, Tailwind CSS for design systems
Backend: Node.js with Express, hosted on Render
Database: PostgreSQL on Supabase with real-time features
Infrastructure: Vercel deployment, Clerk authentication, PostHog analytics

Build for Deep Utility, Not Shallow Engagement

We follow Superhuman's approach to product-market fit: the users who need these tools should be genuinely upset if they disappeared. We're not optimizing for time-on-platform or user countsâ€”we're building tools that amplify human-ness and presence for the people who really need them.

Personal Testimonial Platform

The insight: People need different types of social proof for different contexts, but existing systems are one-size-fits-all. A roommate application needs different testimonials than a job application or dating profile. Plus, people want control over their testimonialsâ€”both giving and receiving them.

The solution: Context-specific testimonial requests and flexible display options. Create different testimonial campaigns for job applications, roommate searches, side hustles, or dating profiles.
